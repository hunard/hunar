What this project does
This project simulates how machines in a factory communicate with each other.

In real life, machines send data using different methods (called protocols), but things like noise or interference can cause errors. This simulator shows how those problems happen and how systems deal with them.

Key features
Uses different communication methods: SPI, I2C, UART
Lets you control noise to see how errors occur
Automatically chooses the best method based on conditions
Retries sending data if it fails
Shows simple stats of success and failure
Why this matters

In real factories, communication failures can stop machines or cause wrong actions.
This project helps understand how systems stay reliable even when things go wrong.
